1、涉及到安全问题，公司内部的app名称、启动入口都已删除；这是内部培训使用的案例，测试人员根据业务要求自由发挥；
2、下拉列表数据使用自定义关键字，因为下拉列表的数据是通过接口请求获取的，默认是5条，每次向下翻动会增加数据，数据量[5,17]，超过17条会覆盖前面的数据，使用传统的获取下拉列表数据方法无法实现，只能自定义；
3、这里使用了两种方式：1、直接在AppiumLibrary关键字增加操作；2、继承AppiumLibrary，这种好处是没有侵入性，服务端不需要重复打包发布；
4、数据驱动、数据库数据校验已删除；
5、这里只是Android 端展示，IOS 见另一版；
